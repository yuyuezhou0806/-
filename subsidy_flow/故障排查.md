# cpolar 闪退 / 连接失败 排查指南

## 现象描述
双击 `start.bat` 后，黑色窗口出现 cpolar 界面（显示 Tunnel Status: connecting），然后窗口突然关闭，没有看到 `Forwarding https://xxx.cpolar.io` 链接。

---

## 排查步骤（按顺序来）

### 第1步：确认 authtoken 已配置

**问题根源**：cpolar 免费版必须先配置 authtoken 才能建立隧道。

**检查方法**：
1. 按 `Win + R`，输入 `cmd`，回车
2. 输入以下命令（注意替换路径）：
```cmd
cd C:\Users\admin\cc_test\subsidy_flow
cpolar.exe authtoken
```
3. 如果显示 `Authtoken not configured`，说明没配置

**解决方法**：
1. 打开 https://dashboard.cpolar.com/auth 注册/登录
2. 左侧菜单点击 **"验证" → "您的 authtoken"**
3. 复制那一串字符（类似 `2abc3def...`）
4. 在 CMD 中运行：
```cmd
cd C:\Users\admin\cc_test\subsidy_flow
cpolar.exe authtoken 你复制的authtoken
```
5. 看到 `Authtoken saved` 表示成功

---

### 第2步：单独测试 cpolar 是否能跑

**先确保本地服务在运行**：
1. 双击 `start_local.bat`（只启动本地服务，不启动穿透）
2. 等3秒后，浏览器打开 `http://localhost:8000`，确认页面能打开
3. **不要关闭这个黑窗口**

**再单独测试 cpolar**：
1. 再开一个 CMD 窗口
2. 输入：
```cmd
cd C:\Users\admin\cc_test\subsidy_flow
cpolar.exe http 8000
```
3. 观察是否出现 `Forwarding https://xxx.cpolar.io -> http://localhost:8000`

如果这里也闪退，看第3步。

---

### 第3步：检查 cpolar 报错信息

闪退太快看不到错误？用手动方式捕获错误：

1. 开一个 CMD 窗口
2. 输入：
```cmd
cd C:\Users\admin\cc_test\subsidy_flow
cpolar.exe http 8000 2>&1 | more
```

这样即使出错，文字也会留在屏幕上。

**常见错误及解决**：

| 错误提示 | 原因 | 解决 |
|---------|------|------|
| `Authtoken not configured` | 没配置token | 按第1步配置 |
| `account not subscribed` / `upgrade required` | 免费额度用完 | 等1小时再试，或购买付费版 |
| `dial tcp: connection refused` | 8000端口没服务 | 先启动 `start_local.bat` |
| `cannot connect to server` | 网络不通 | 检查能否打开 cpolar.com |
| `ERR_NGROK_702` | 同时开了太多隧道 | 关闭其他 cpolar 进程 |

---

### 第4步：关闭旧进程再试

可能之前有残留的 cpolar 进程在跑，导致新的启动失败。

1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 找到所有 `cpolar.exe` 和 `python.exe`，全部结束任务
3. 重新双击 `start.bat`

---

### 第5步：换 ngrok 试试（cpolar 的替代品）

如果 cpolar 实在用不了，可以换 ngrok，同样免费。

**安装步骤**：
1. 访问 https://ngrok.com/download
2. 下载 Windows 64-bit 版本
3. 解压后把 `ngrok.exe` 放到 `subsidy_flow` 文件夹里
4. 访问 https://dashboard.ngrok.com/get-started/your-authtoken 复制 token
5. CMD 中运行：`ngrok.exe authtoken 你的token`
6. 启动本地服务（`start_local.bat`）
7. 再开一个 CMD，运行：`ngrok.exe http 8000`
8. 会显示 `Forwarding https://xxxx.ngrok-free.app -> http://localhost:8000`

**ngrok 缺点**：免费版域名每次也变，每次重启要更新链接。但稳定性通常比 cpolar 好一些。

---

### 第6步：局域网先用着（不追求公网）

如果公网穿透搞不定，至少保证办公室内部能用：

1. 双击 `start_local.bat`
2. 在同一局域网的电脑上，浏览器打开 `http://运行电脑的IP:8000`
3. 查看本机IP的方法：运行 `start_local.bat` 后，黑窗口会显示局域网地址

给客户展示时，可以让客户到你们办公室来，连你们的 WiFi 访问。
