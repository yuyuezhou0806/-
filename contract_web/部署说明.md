# 合同生成器 · Web 内网版

公司内部同事用浏览器(电脑 / 手机)上传项目网页截图,自动生成 Word 检测合同。

## 怎么用

### 启动服务(管理员一次性操作)

1. 双击 `start.bat`
2. 弹出黑色控制台窗口,显示"本机 / 内网"两个 URL
3. **保持窗口开着**(关掉就停服务)

### 访问(任何同事)

电脑 / 手机浏览器打开内网 URL(形如 `http://192.168.x.x:8000/`):

1. 把项目截图拖到上传区(或者点击选择文件)
2. 等 3-5 秒
3. 浏览器自动下载 `<工程名>-已填充.doc`
4. 页面上展示识别到的字段供核对

### 截图要求

来源**上海市建设工程项目信息报送系统**,清晰可读,至少包含:
- 工程名称
- 建设单位
- 施工单位
- 建设地点 / 报建编号 / 建筑面积

## 依赖

由 `..\.venv` 提供:
- `fastapi`、`uvicorn`(Web 框架)
- `rapidocr_onnxruntime`(OCR 引擎,不依赖 paddle)
- `pywin32`(调 Word COM 接口)
- `python-docx`(可选)

**必须**:这台电脑装了 **Microsoft Word**(WPS 不行)。

## 网络

- 端口:8000
- 绑定 0.0.0.0 → 局域网可访问
- **第一次启动 Windows 防火墙会弹窗**,选"允许"(私有网络 + 公用网络都勾)

## 常见问题

**问**:同事说访问不了 URL?
- 确认你的 `start.bat` 窗口还开着(没关)
- 确认你电脑和同事在同一个 WiFi / 同一个内网
- 检查 Windows 防火墙没拦 Python(允许 `python.exe` 通过私有/公用网络)

**问**:第一次上传超慢?
- RapidOCR 首次会加载模型(几 MB),只需 1-3 秒。比 paddle 快很多。
- 第二次开始立即响应。

**问**:报"未提取到关键字段"?
- 截图可能不是上海市建设工程项目信息报送系统页面
- 或者截图分辨率太低
- 失败页面会显示 OCR 识别到的文字,可以贴出来给开发同学诊断

**问**:Word 报"宏警告 / 启用编辑"?
- 是 win32com 操作 Word 的正常现象,允许即可
