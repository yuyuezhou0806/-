## Tier 2 · 重要(让 Agent 专业度暴涨)

这些**只能从你公司内部拿**——但拿到后效果远超公开规范,因为是真实项目语料。

| 材料 | 数量建议 | 用途 |
|------|---------|------|
| 历史合同样本(已脱敏) | 20-50 份 | Agent 知道合同条款的真实表述 |
| 历史检测报告模板 | 5-10 份 | Agent 知道报告结构 |
| 项目类型 → 检测项映射表 | 1 份 | Agent 推荐检测项的核心依据 |
| 公司内部 SOP / 操作规程 | 全要 | Agent 知道公司怎么做事 |
| 费率表 / 报价标准 | 1 份 | Agent 估算费用 |
| 常见 Q&A 文档 | 越多越好 | 同事常问的问题答案 |

**拿这些前要做的**:
- 跟领导沟通,说明这是内部用 Agent,不外发
- 客户名、项目地址、金额这类敏感字段做脱敏(用占位符)
- 涉及具体人名 → `XX 工程师`

## Tier 3 · 加分(让 Agent 像老专家)

| 材料 | 用途 |
|------|------|
| 已有的 IDI 缺陷库 599 条 | **已经在 `../idi_defects/data/defects.json`,直接复用** ✓ |
| 检测案例报告(已脱敏) | 真实项目结果,Agent 能学到判断逻辑 |
| 《建筑工程检测技术手册》系列书 | 行业通识 |
| 不合格案例集 | Agent 知道什么情况是问题 |
| 其他省市地方标准 | 跨地区项目用 |
